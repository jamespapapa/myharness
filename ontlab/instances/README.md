# instances/

각 target repository는 여기서 독립적인 ontology instance를 가진다.

예:
- `instances/my-service/graph/`
- `instances/my-service/evidence/`
- `instances/my-service/questions/`
- `instances/my-service/answers/`
- `instances/my-service/eval/`
- `instances/my-service/reports/`

주의:
- 서로 다른 repo의 fact를 섞지 않는다.
- seed repo 데이터는 여기에 저장하지 않는다.
